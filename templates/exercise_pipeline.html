Exercise Pipeline

1. Dataset Inspection

The training set contains 1,460 rows and 81 columns, loaded from train.csv alongside test.csv. The inspection phase covers:

  • Column non-null counts and dtype breakdown via .info()
  • Descriptive statistics (count, mean, std, min/max, percentiles) via .describe()
  • Target variable distribution — a histogram of SalePrice confirms right skew
  • Pearson correlation between SalePrice and all numeric columns; the top 10 correlated features are extracted and visualised in a heatmap

Top correlated features include OverallQual, GrLivArea, GarageCars, GarageArea, and TotalBsmtSF.

2. Outlier & Missing-Value Handling

Two clear outliers are identified from a GrLivArea vs SalePrice scatter plot — properties larger than 4,300 sq ft but priced below $200,000 — and are removed from the training set.

Missing values are handled as follows:

  • Columns with NaN counts and percentages are printed and ranked
  • Categorical NaNs are filled with "None" (e.g. PoolQC NaN means no pool)
  • Numerical NaNs: GarageYrBlt is set to 0; all other numeric columns are filled with their column mean

Finally, all remaining categorical columns are one-hot encoded with pd.get_dummies(drop_first=True), converting the dataset to an entirely numeric representation.

3. Feature Engineering & Normalisation

The target SalePrice is separated from the feature matrix. Three highly collinear area columns are merged into a single composite feature:

TotalSF = TotalBsmtSF + 1stFlrSF + 2ndFlrSF

The original three columns are then dropped, reducing redundancy.

All numeric features are standardised with StandardScaler (zero mean, unit variance). The target is log-transformed with np.log1p(SalePrice) to reduce skew and stabilise variance, making it suitable for linear modelling.

4. Model Selection, Training & Persistence

A Ridge Regression model (alpha=10.0) is selected for its robustness against multicollinearity in high-dimensional one-hot encoded data.

Model performance is evaluated using 5-fold cross-validation with neg_mean_squared_error scoring. RMSE is computed per fold and averaged:

rmse_scores = np.sqrt(-cross_val_score(model, x_train, y_train, cv=5, scoring='neg_mean_squared_error'))

The final model is then trained on the full clean training set and persisted to disk with joblib:

  • ridge_model.pkl — trained Ridge model
  • scaler.pkl — fitted StandardScaler
  • model_columns.pkl — ordered feature column names

At startup, the API loads these artefacts and uses them to serve predictions at the POST /predict endpoint.
